How API Integration Works

The Attio CRM API is a RESTful interface that allows external services to interact with your CRM data. With the API, you can automate processes and synchronize data in real-time.

1. Enable the API and Get Credentials

To connect your application, you need the **App ID** and the **Application Access Key**. Follow these steps:

  1. Sign in to Attio and navigate to your CRM project.
  2. Go to `Settings` > `API Integrations`.
  3. In the `API Access` section, activate **`Enable`**.
  4. Copy the **`App ID`** and create an **`API Access Key`**. Store this key securely.

2. API URL Structure

All requests are directed to a base URL that includes your App ID and the data type you want to interact with.

https://api.attio.com/api/v2/apps/{appId}/data/{dataType}/Action
  • {appId}: Your unique application ID.
  • {dataType}: The type of data you want to interact with (e.g., `Leads`, `Contacts`, `Accounts`).
  • Action: The operation to perform (e.g., `Create`, `Read`, `Update`, `Delete`).

3. Making an API Request

Each request must include two main headers and a JSON body for the operation.

Required Headers:

Content-Type: application/json
ApplicationAccessKey: YOUR_ACCESS_KEY

Example Request (Creating a new lead):

To add a new lead, the request body should have this format:

{
  "Action": "Create",
  "Properties": {},
  "Data": [
    {
      "Name": "John Doe",
      "Email": "john.doe@example.com",
      "Source": "Website"
    }
  ]
}

Ensure that the field names (`"Name"`, `"Email"`, etc.) exactly match those in your Attio CRM.

4. Troubleshooting

If the request fails, check the following points:

  • Verify your credentials (`App ID` and `API Access Key`).
  • Confirm that your JSON syntax is correct.
  • Make sure the API is enabled in your Attio CRM settings.
